# Task Logs — Engineering Journal

What we built, how, and why. Read these when asking "what did we do for Task X?" or "why did we choose approach A over B?"

**Not for:** documenting how external tools/APIs work — that goes in `notes/`.

## Structure

One file per completed task: `task_2_1_gp_data_fetcher.md`, `task_2_2_coordinate_transforms.md`, etc.

Each log documents:
- **Date** — when the task was completed
- **Goal** — what we set out to build and why
- **Approach** — how we tackled it, what alternatives were considered
- **Findings** — discoveries, surprises, gotchas encountered during implementation
- **Results** — what was built, test outcomes, validated behavior
- **Files** — what was created or modified
- **Function reference** — how the code works at a glance
- **Lessons learned** — what to carry forward to future tasks
